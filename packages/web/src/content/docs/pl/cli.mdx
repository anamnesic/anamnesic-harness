CLI kairos domyślnie uruchamia [TUI](/docs/tui), gdy wywołasz je bez argumentów.

```bash
kairos
```

CLI obsługuje też polecenia opisane na tej stronie, więc możesz używać kairos programowo.

```bash
kairos run "Explain how closures work in JavaScript"
```

---

### tui

Uruchamia interfejs terminalowy (TUI) kairos.

```bash
kairos [project]
```

#### Flagi

| Flaga        | Skrót | Opis                                                                    |
| ------------ | ----- | ----------------------------------------------------------------------- |
| `--continue` | `-c`  | Kontynuuj ostatnią sesję                                                |
| `--session`  | `-s`  | Identyfikator sesji do kontynuowania                                    |
| `--fork`     |       | Sklonuj sesję podczas kontynuacji (użyj z `--continue` lub `--session`) |
| `--prompt`   |       | Monit do użycia                                                         |
| `--model`    | `-m`  | Model do użycia w formacie dostawca/model                               |
| `--agent`    |       | Agent do użycia                                                         |
| `--port`     |       | Port do nasłuchiwania                                                   |
| `--hostname` |       | Nazwa hosta, do której należy się powiązać                              |

## Polecenia

Interfejs CLI kairos zawiera także dodatkowe polecenia.

---

### agent

Zarządzaj agentami kairos.

```bash
kairos agent [command]
```

---

### attach

Dołącz terminal do już działającego serwera backendu kairos uruchomionego za pomocą `serve` lub `web`.

```bash
kairos attach [url]
```

Pozwala to połączyć TUI ze zdalnym backendem kairos. Na przykład:

```bash
# Start the backend server for web/mobile access
kairos web --port 4096 --hostname 0.0.0.0

# In another terminal, attach the TUI to the running backend
kairos attach http://10.20.30.40:4096
```

#### Flagi

| Flaga       | Skrót | Opis                                    |
| ----------- | ----- | --------------------------------------- |
| `--dir`     |       | Katalog roboczy, w którym uruchomić TUI |
| `--session` | `-s`  | Identyfikator sesji do kontynuowania    |

---

#### create

Utwórz nowego agenta z niestandardową konfiguracją.

```bash
kairos agent create
```

To polecenie poprowadzi Cię przez proces tworzenia nowego agenta z niestandardowym promptem systemowym i konfiguracją narzędzi.

---

#### list

Wyświetl listę wszystkich agentów.

```bash
kairos agent list
```

---

### auth

Zarządzaj danymi uwierzytelniającymi i logowaniem dla dostawców.

```bash
kairos auth [command]
```

---

#### login

kairos integruje się z wieloma dostawcami. Możesz użyć `kairos auth login`, aby skonfigurować klucz API dla dowolnego obsługiwanego dostawcy. Klucze są bezpiecznie przechowywane w `~/.local/share/kairos/auth.json`.

```bash
kairos auth login
```

Podczas uruchamiania kairos ładuje dostawców z pliku poświadczeń, a także ze zmiennych środowiskowych lub pliku `.env` w Twoim projekcie.

---

#### list

Wyświetl listę wszystkich uwierzytelnionych dostawców przechowywanych w pliku poświadczeń.

```bash
kairos auth list
```

Lub skrócona wersja:

```bash
kairos auth ls
```

---

#### logout

Wyloguj się z dostawcy, usuwając go z pliku poświadczeń.

```bash
kairos auth logout
```

### github

```bash

```

#### install

Zainstaluj agenta GitHub w swoim repozytorium.

```bash
kairos github install
```

Spowoduje to utworzenie pliku workflow GitHub Actions i poprowadzi Cię przez proces konfiguracji. [Dowiedz się więcej](/docs/github).


##### Flagi

| Flaga     | Opis                                     |
| --------- | ---------------------------------------- |
| `--event` | Zdarzenie GitHub, które wyzwoliło agenta |
| `--token` | Osobisty token dostępu GitHub            |

### mcp

Zarządzaj serwerami Model Context Protocol (MCP).

```bash
kairos mcp [command]
```

---

#### add

Dodaj serwer MCP do swojej konfiguracji.

```bash
kairos mcp add
```

To polecenie poprowadzi Cię przez proces dodawania lokalnego lub zdalnego serwera MCP.

---

#### list

Wyświetl listę wszystkich skonfigurowanych serwerów MCP i ich status.

```bash
kairos mcp list
```

Lub skrócona wersja:

```bash
kairos mcp ls
```

---

#### auth

Uwierzytelnij się w serwerze MCP z włączoną funkcją OAuth.

```bash
kairos mcp auth [name]
```

Jeśli nie podasz nazwy serwera, zostaniesz poproszony o wybranie jednego z serwerów wymagających uwierzytelnienia.

Możesz także wyświetlić listę serwerów obsługujących OAuth i ich status uwierzytelnienia.

```bash
kairos mcp auth list
```

Lub skrócona wersja:

```bash
kairos mcp auth ls
```

---

#### logout

Usuń tokeny OAuth dla serwera MCP.

```bash
kairos mcp logout [name]
```

---

#### debug

Debuguj problemy z połączeniem OAuth dla serwera MCP.

```bash
kairos mcp debug <name>
```

### models

Wyświetl listę modeli ze wszystkich skonfigurowanych dostawców.

```bash
kairos models [provider]
```

Pokazuje modele dostępne do użycia w `provider/model`.

Jest to identyfikator modelu, którego używasz w [konfiguracji](/docs/config).

Opcjonalnie możesz podać identyfikator dostawcy, aby filtrować modele według tego dostawcy.

```bash
kairos models anthropic
```

#### Flagi

| Flaga       | Opis                                                                            |
| ----------- | ------------------------------------------------------------------------------- |
| `--refresh` | Odśwież pamięć podręczną modeli                                                 |
| `--verbose` | Bardziej szczegółowe dane wyjściowe modelu (zawiera metadane, takie jak koszty) |

Użyj flagi `--refresh`, aby zaktualizować listę modeli w pamięci podręcznej. Jest to przydatne, gdy dostawca dodał nowe modele, które chcesz zobaczyć w kairos.

```bash
kairos models --refresh
```

### run

Uruchom kairos w trybie nieinteraktywnym, przekazując bezpośrednio prompt.

```bash
kairos run [message..]
```

Jest to przydatne do tworzenia skryptów, automatyzacji lub gdy chcesz wysłać szybkie zapytanie bez uruchamiania TUI. Na przykład:

```bash "kairos run"
kairos run Explain the use of context in Go
```

Można również użyć tego z działającym `kairos serve`, aby uniknąć zimnego startu serwera MCP przy każdym uruchomieniu:

```bash
# Start a headless server in one terminal
kairos serve

# In another terminal, run commands that attach to it
kairos run --attach http://localhost:4096 "Explain async/await in JavaScript"
```

#### Flagi

| Flaga        | Skrót | Opis                                                                    |
| ------------ | ----- | ----------------------------------------------------------------------- |
| `--command`  |       | Polecenie do uruchomienia, reszta to argumenty                          |
| `--continue` | `-c`  | Kontynuuj ostatnią sesję                                                |
| `--session`  | `-s`  | Identyfikator sesji do kontynuowania                                    |
| `--fork`     |       | Sklonuj sesję podczas kontynuacji (użyj z `--continue` lub `--session`) |
| `--share`    |       | Udostępnij sesję po zakończeniu                                         |
| `--model`    | `-m`  | Model do użycia w formacie dostawca/model                               |
| `--agent`    |       | Agent do użycia                                                         |
| `--file`     | `-f`  | Pliki do załączenia do wiadomości                                       |
| `--format`   |       | Format wyjściowy: `default` (sformatowany) lub `json` (surowy JSON)     |
| `--title`    |       | Tytuł sesji (jeśli nie podano, zostanie wygenerowany z promptu)         |
| `--attach`   |       | Dołącz do działającego serwera kairos (np. http://localhost:4096)     |
| `--port`     |       | Port dla serwera lokalnego (domyślnie losowy)                           |

### serve

Uruchom serwer kairos (bez interfejsu) w celu uzyskania dostępu do API. Pełny opis API HTTP znajduje się w [dokumentacji serwera](/docs/server).

```bash
kairos serve
```

Uruchamia to serwer HTTP, który zapewnia dostęp do API kairos bez interfejsu TUI. Ustaw `kairos_SERVER_PASSWORD`, aby włączyć podstawowe uwierzytelnianie HTTP (domyślna nazwa użytkownika to `kairos`).

#### Flagi

| Flaga        | Opis                                       |
| ------------ | ------------------------------------------ |
| `--port`     | Port do nasłuchiwania                      |
| `--hostname` | Nazwa hosta, do której należy się powiązać |
| `--mdns`     | Włącz wykrywanie mDNS                      |
| `--cors`     | Dodatkowe dozwolone źródła CORS            |

### stats

Pokaż statystyki wykorzystania tokenów i sesji kairos.

```bash
kairos stats
```

#### Flagi

| Flaga       | Opis                                                                                  |
| ----------- | ------------------------------------------------------------------------------------- |
| `--days`    | Pokaż statystyki z ostatnich N dni (domyślnie: cały czas)                             |
| `--tools`   | Pokaż użycie poszczególnych narzędzi (domyślnie: wszystkie)                           |
| `--models`  | Pokaż podział na modele (domyślnie ukryty). Podaj liczbę, aby pokazać N najczęstszych |
| `--project` | Filtruj według projektu (domyślnie: wszystkie projekty, pusty ciąg: bieżący projekt)  |

### export

Eksportuj dane sesji jako JSON.

```bash
kairos export [sessionID]
```

Jeśli nie podasz identyfikatora sesji, zostaniesz poproszony o wybranie jednej z ostatnich sesji.

---

### import

Importuj dane sesji z pliku JSON lub adresu URL udostępniania kairos.

```bash
kairos import <file>
```

Możesz importować z pliku lokalnego lub adresu URL.

```bash
kairos import session.json
kairos import https://opncd.ai/s/abc123
```

### web

Uruchom serwer kairos z interfejsem internetowym.

```bash
kairos web
```

Uruchamia to serwer HTTP i udostępnia kairos przez interfejs przeglądarkowy. Ustaw `kairos_SERVER_PASSWORD`, aby włączyć podstawowe uwierzytelnianie HTTP (domyślna nazwa użytkownika to `kairos`).

#### Flagi

| Flaga        | Opis                                       |
| ------------ | ------------------------------------------ |
| `--port`     | Port do nasłuchiwania                      |
| `--hostname` | Nazwa hosta, do której należy się powiązać |
| `--mdns`     | Włącz wykrywanie mDNS                      |
| `--cors`     | Dodatkowe dozwolone źródła CORS            |

### acp

Uruchom serwer ACP (Agent Client Protocol).

```bash
kairos acp
```

Uruchamia serwer ACP, który komunikuje się przez stdin/stdout przy użyciu JSON-RPC.

#### Flagi

| Flaga        | Opis                                       |
| ------------ | ------------------------------------------ |
| `--cwd`      | Katalog roboczy                            |
| `--port`     | Port do nasłuchiwania                      |
| `--hostname` | Nazwa hosta, do której należy się powiązać |

### uninstall

Odinstaluj kairos i usuń wszystkie powiązane pliki.

```bash
kairos uninstall
```

#### Flagi

| Flaga           | Skrót | Opis                          |
| --------------- | ----- | ----------------------------- |
| `--keep-config` | `-c`  | Zachowaj pliki konfiguracyjne |
| `--keep-data`   | `-d`  | Zachowaj dane sesji i migawki |
| `--dry-run`     |       | Pokaż co zostanie usunięte    |
| `--force`       | `-f`  | Pomiń monity o potwierdzenie  |

---

### upgrade

Aktualizuj kairos do najnowszej lub określonej wersji.

```bash
kairos upgrade [target]
```

Aby zaktualizować do najnowszej wersji:

```bash
kairos upgrade
```

Aby zaktualizować do konkretnej wersji:

```bash
kairos upgrade v0.1.48
```

#### Flagi

| Flaga      | Skrót | Opis                                                |
| ---------- | ----- | --------------------------------------------------- |
| `--method` | `-m`  | Wymuś metodę instalacji: curl, npm, pnpm, bun, brew |

## Zmienne środowiskowe

kairos można skonfigurować za pomocą zmiennych środowiskowych.

| Zmienna                               | Typ     | Opis                                                       |
| ------------------------------------- | ------- | ---------------------------------------------------------- |
| `kairos_AUTO_SHARE`                 | boolean | Automatycznie udostępniaj sesje                            |
| `kairos_GIT_BASH_PATH`              | string  | Ścieżka do pliku wykonywalnego Git Bash w systemie Windows |
| `kairos_CONFIG`                     | string  | Ścieżka do pliku konfiguracyjnego                          |
| `kairos_TUI_CONFIG`                 | string  | Ścieżka do pliku konfiguracyjnego TUI                      |
| `kairos_CONFIG_DIR`                 | string  | Ścieżka do katalogu konfiguracyjnego                       |
| `kairos_CONFIG_CONTENT`             | string  | Treść konfiguracji JSON (inline)                           |
| `kairos_DISABLE_AUTOUPDATE`         | boolean | Wyłącz automatyczne sprawdzanie aktualizacji               |
| `kairos_DISABLE_PRUNE`              | boolean | Wyłącz czyszczenie starych wyników (pruning)               |
| `kairos_DISABLE_TERMINAL_TITLE`     | boolean | Wyłącz automatyczne ustawianie tytułu terminala            |
| `kairos_PERMISSION`                 | string  | Konfiguracja uprawnień w JSON (inline)                     |
| `kairos_DISABLE_DEFAULT_PLUGINS`    | boolean | Wyłącz domyślne wtyczki                                    |
| `kairos_DISABLE_LSP_DOWNLOAD`       | boolean | Wyłącz automatyczne pobieranie serwerów LSP                |
| `kairos_ENABLE_EXPERIMENTAL_MODELS` | boolean | Włącz modele eksperymentalne                               |
| `kairos_DISABLE_AUTOCOMPACT`        | boolean | Wyłącz automatyczne kompaktowanie kontekstu                |
| `kairos_DISABLE_kairos_CODE`        | boolean | Wyłącz integrację z `.kairos` (prompt + skills)            |
| `kairos_DISABLE_kairos_CODE_PROMPT` | boolean | Wyłącz czytanie `~/.kairos/kairos.md`                      |
| `kairos_DISABLE_kairos_CODE_SKILLS` | boolean | Wyłącz ładowanie `.kairos/skills`                          |
| `kairos_DISABLE_MODELS_FETCH`       | boolean | Wyłącz pobieranie modeli ze źródeł zewnętrznych            |
| `kairos_FAKE_VCS`                   | string  | Fałszywy dostawca VCS do celów testowych                   |
| `kairos_CLIENT`                     | string  | Identyfikator klienta (domyślnie `cli`)                    |
| `kairos_ENABLE_EXA`                 | boolean | Włącz narzędzie wyszukiwania internetowego Exa             |
| `kairos_SERVER_PASSWORD`            | string  | Włącz uwierzytelnianie podstawowe dla `serve`/`web`        |
| `kairos_SERVER_USERNAME`            | string  | Nazwa użytkownika do autoryzacji (domyślnie `kairos`)    |
| `kairos_MODELS_URL`                 | string  | Niestandardowy adres URL do pobierania konfiguracji modeli |

| `kairos_EXPERIMENTAL_DISABLE_FILEWATCHER`     | boolean | Wyłącz obserwatora plików                    |
| `kairos_EXPERIMENTAL_EXA`                     | boolean | Włącz funkcje eksperymentalne Exa            |
| `kairos_EXPERIMENTAL_LSP_TY`                  | boolean | Włącz TY LSP dla plików python               |
| `kairos_EXPERIMENTAL_MARKDOWN`                | boolean | Włącz funkcje eksperymentalne Markdown       |

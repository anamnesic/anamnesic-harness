kairos CLI starter som standard [TUI](/docs/tui), nĂ„r den kĂžrer uden nogen argumenter.

```bash
kairos
```

Men den accepterer ogsĂ„ kommandoer som dokumenteret pĂ„ denne side. Dette giver dig mulighed for at interagere med kairos programmatisk.

```bash

```

### tui

Start kairos terminalbrugergrĂŠnsefladen.

```bash
kairos [project]
```

#### Flag

| Flag         | Kort | Beskrivelse                                                                  |
| ------------ | ---- | ---------------------------------------------------------------------------- |
| `--continue` | `-c` | FortsĂŠt sidste session                                                       |
| `--session`  | `-s` | Sessions-id for at fortsĂŠtte                                                 |
| `--fork`     |      | Forgren sessionen ved fortsĂŠttelse (brug med `--continue` eller `--session`) |
| `--prompt`   |      | SpĂžrg om at bruge                                                            |
| `--model`    | `-m` | Model til brug i form af provider/model                                      |
| `--agent`    |      | Agent hos bruge                                                              |
| `--port`     |      | Port at lytte pĂ„                                                             |
| `--hostname` |      | VĂŠrtsnavn at lytte pĂ„                                                        |

## Kommandoer

kairos CLI har ogsĂ„ fĂžlgende kommandoer.

---

### agent

Administrer agenter i kairos.

```bash
kairos agent [command]
```

### attach

Tilslut en terminal til en allerede kĂžrende kairos backend-server startet via `serve` eller `web` kommandoer.

```bash
kairos attach [url]
```

Dette gĂžr det muligt at bruge TUI med en ekstern kairos backend. For eksempel:

```bash
# Start the backend server for web/mobile access
kairos web --port 4096 --hostname 0.0.0.0

# In another terminal, attach the TUI to the running backend
kairos attach http://10.20.30.40:4096
```

#### Flag

| Flag        | Kort | Beskrivelse                      |
| ----------- | ---- | -------------------------------- |
| `--dir`     |      | Arbejdsmappe til at starte TUI i |
| `--session` | `-s` | Sessions-id for at fortsĂŠtte     |

### auth

Kommando til at administrere legitimationsoplysninger og login for udbydere.

```bash
kairos auth [command]
```

---

#### login

kairos drives af udbyderlisten pĂ„ [Models.dev](https://models.dev), sĂ„ du kan bruge `kairos auth login` til at konfigurere API-nĂžgler for enhver udbyder, du vil bruge. Dette er gemt i `~/.local/share/kairos/auth.json`.

```bash
kairos auth login
```

NĂ„r kairos starter op, indlĂŠser den udbyderne fra legitimationsfilen. Og hvis der er nogen nĂžgler defineret i dine miljĂžer eller en `.env` fil i dit projekt.

---

#### list

Viser alle de godkendte udbydere som gemt i legitimationsfilen.

```bash
kairos auth list
```

Eller den korte version.

```bash
kairos auth ls
```

---

#### logout

Logger dig ud af en udbyder ved at rydde den fra legitimationsfilen.

```bash
kairos auth logout
```


#### add


### models

Liste over alle tilgĂŠngelige modeller fra konfigurerede udbydere.

```bash
kairos models [provider]
```

Denne kommando viser alle tilgĂŠngelige modeller pĂ„ tvĂŠrs af dine konfigurerede udbydere i formatet `provider/model`.

Dette er nyttigt til at finde ud af det nĂžjagtige modelnavn, der skal bruges i [your config](/docs/config/).

Du kan eventuelt videregive et udbyder-id for at filtrere modeller efter den udbyder.

```bash
kairos models anthropic
```

#### Flag

| Flag        | Beskrivelse                                                             |
| ----------- | ----------------------------------------------------------------------- |
| `--refresh` | Opdater modelcachen fra models.dev                                      |
| `--verbose` | Brug mere detaljeret modeloutput (inkluderer metadata som omkostninger) |

Brug flaget `--refresh` til at opdatere den cachelagrede modelliste. Dette er nyttigt, nĂ„r nye modeller er blevet tilfĂžjet til en udbyder, og du vil se dem i kairos.

```bash
kairos models --refresh
```

### run

KĂžr kairos i ikke-interaktiv tilstand ved at sende en prompt direkte.

```bash
kairos run [message..]
```

Dette er nyttigt til scripting, automatisering, eller nĂ„r du vil have et hurtigt svar uden at starte hele TUI. F.eks.

```bash "kairos run"

```

Du kan ogsĂ„ vedhĂŠfte til en kĂžrende `kairos serve`-instans for at undgĂ„ MCP serverens kolde opstartstider ved hver kĂžrsel:

```bash
# Start a headless server in one terminal
kairos serve

# In another terminal, run commands that attach to it
kairos run --attach http://localhost:4096 "Explain async/await in JavaScript"
```

#### Flag

| Flag         | Kort | Beskrivelse                                                                         |
| ------------ | ---- | ----------------------------------------------------------------------------------- |
| `--command`  |      | Kommandoen til at kĂžre, brug besked til args                                        |
| `--continue` | `-c` | FortsĂŠt sidste session                                                              |
| `--session`  | `-s` | Sessions-id for at fortsĂŠtte                                                        |
| `--fork`     |      | Forgren sessionen ved fortsĂŠttelse (brug med `--continue` eller `--session`)        |
| `--share`    |      | Del sessionen                                                                       |
| `--model`    | `-m` | Model til brug i form af provider/model                                             |
| `--agent`    |      | Agent til brug                                                                      |
| `--file`     | `-f` | Fil(er), der skal vedhĂŠftes til meddelelsen                                         |
| `--format`   |      | Format: standard (formateret) eller json (rĂ„ JSON hĂŠndelser)                        |
| `--title`    |      | Titel for sessionen (bruger trunkeret prompt, hvis der ikke er angivet nogen vĂŠrdi) |
| `--attach`   |      | Tilslut til en kĂžrende kairos-server (f.eks. http://localhost:4096)               |
| `--port`     |      | Port til den lokale server (standard til vilkĂ„rlig port)                            |

### serve

Start en hovedlĂžs kairos-server til API-adgang. Tjek [server docs](/docs/server) for den fulde HTTP-grĂŠnseflade.

```bash

```

Dette starter en HTTP-server, der giver API-adgang til kairos-funktionalitet uden TUI-grĂŠnsefladen. Indstil `kairos_SERVER_PASSWORD` for at aktivere HTTP grundlĂŠggende godkendelse (brugernavn er standard til `kairos`).

#### Flag

| Flag         | Beskrivelse                                      |
| ------------ | ------------------------------------------------ |
| `--port`     | Port at lytte pĂ„                                 |
| `--hostname` | VĂŠrtsnavn at lytte pĂ„                            |
| `--mdns`     | Aktiver mDNS-opdagelse                           |
| `--cors`     | Yderligere browseroprindelse til at tillade CORS |

### import


### upgrade

Opdaterer kairos til den nyeste version eller en specifik version.

```bash

```

For at opgradere til den nyeste version.

```bash

```

For at opgradere til en bestemt version.

```bash

```

#### upgrade

| Flag       | Kort | Beskrivelse                                                      |
| ---------- | ---- | ---------------------------------------------------------------- |
| `--method` | `-m` | Installationsmetoden, der blev brugt; curl, npm, pnpm, bun, brew |

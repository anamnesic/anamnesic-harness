### Instalacja

```bash
# YOLO
curl -fsSL https://kairos.ai/install | bash

# Menedżery pakietów
npm i -g kairos-ai@latest        # albo bun/pnpm/yarn
scoop install kairos             # Windows
choco install kairos             # Windows
brew install anomalyco/tap/kairos # macOS i Linux (polecane, zawsze aktualne)
brew install kairos              # macOS i Linux (oficjalna formuła brew, rzadziej aktualizowana)
sudo pacman -S kairos            # Arch Linux (Stable)
paru -S kairos-bin               # Arch Linux (Latest from AUR)
mise use -g kairos               # dowolny system
nix run nixpkgs#kairos           # lub github:anomalyco/kairos dla najnowszej gałęzi dev
```

> [!TIP]
> Przed instalacją usuń wersje starsze niż 0.1.x.

### Aplikacja desktopowa (BETA)

kairos jest także dostępny jako aplikacja desktopowa. Pobierz ją bezpośrednio ze strony [releases](https://github.com/anomalyco/kairos/releases) lub z [kairos.ai/download](https://kairos.ai/download).

| Platforma             | Pobieranie                            |
| --------------------- | ------------------------------------- |
| macOS (Apple Silicon) | `kairos-desktop-darwin-aarch64.dmg` |
| macOS (Intel)         | `kairos-desktop-darwin-x64.dmg`     |
| Windows               | `kairos-desktop-windows-x64.exe`    |
| Linux                 | `.deb`, `.rpm` lub AppImage           |

```bash
# macOS (Homebrew)
brew install --cask kairos-desktop
# Windows (Scoop)
scoop bucket add extras; scoop install extras/kairos-desktop
```

#### Katalog instalacji

Skrypt instalacyjny stosuje następujący priorytet wyboru ścieżki instalacji:

1. `$kairos_INSTALL_DIR` - Własny katalog instalacji
2. `$XDG_BIN_DIR` - Ścieżka zgodna ze specyfikacją XDG Base Directory
3. `$HOME/bin` - Standardowy katalog binarny użytkownika (jeśli istnieje lub można go utworzyć)
4. `$HOME/.kairos/bin` - Domyślny fallback

```bash
# Przykłady
kairos_INSTALL_DIR=/usr/local/bin curl -fsSL https://kairos.ai/install | bash
XDG_BIN_DIR=$HOME/.local/bin curl -fsSL https://kairos.ai/install | bash
```

### Agents

kairos zawiera dwóch wbudowanych agentów, między którymi możesz przełączać się klawiszem `Tab`.

- **build** - Domyślny agent z pełnym dostępem do pracy developerskiej
- **plan** - Agent tylko do odczytu do analizy i eksploracji kodu
  - Domyślnie odmawia edycji plików
  - Pyta o zgodę przed uruchomieniem komend bash
  - Idealny do poznawania nieznanych baz kodu lub planowania zmian

Dodatkowo jest subagent **general** do złożonych wyszukiwań i wieloetapowych zadań.
Jest używany wewnętrznie i można go wywołać w wiadomościach przez `@general`.

Dowiedz się więcej o [agents](https://kairos.ai/docs/agents).

### FAQ

#### Czym to się różni od kairos Code?

Jest bardzo podobne do kairos Code pod względem możliwości. Oto kluczowe różnice:

- 100% open source
- Niezależne od dostawcy. Chociaż polecamy modele oferowane przez [kairos Zen](https://kairos.ai/zen); kairos może być używany z kairos, OpenAI, Google, a nawet z modelami lokalnymi. W miarę jak modele ewoluują, różnice będą się zmniejszać, a ceny spadać, więc ważna jest niezależność od dostawcy.
- Wbudowane wsparcie LSP
- Skupienie na TUI. kairos jest budowany przez użytkowników neovim i twórców [terminal.shop](https://terminal.shop); przesuwamy granice tego, co jest możliwe w terminalu.
- Architektura klient/serwer. Pozwala np. uruchomić kairos na twoim komputerze, a sterować nim zdalnie z aplikacji mobilnej. To znaczy, że frontend TUI jest tylko jednym z możliwych klientów.
